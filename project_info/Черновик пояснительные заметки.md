# Черновик пояснительной записки к проекту Chart Detective

## 1. Введение

Chart Detective - это учебный веб-проект в формате музыкально-географической игры. Пользователь слушает популярные треки из музыкальных чартов разных стран и пытается определить, какой стране принадлежит текущий чарт.

Проект объединяет несколько направлений разработки: серверную часть на Python и Flask, работу с базой данных, авторизацию пользователей, интеграцию с внешними API, обработку JSON-данных и создание интерактивного интерфейса на JavaScript.

Главная идея проекта заключается в том, чтобы сделать игру, где музыка используется как источник подсказок. Пользователь анализирует язык треков, исполнителей, стиль музыки и популярность песен, после чего выбирает страну.

## 2. Цель и задачи проекта

Цель проекта - разработать веб-приложение, позволяющее пользователю угадывать страны по музыкальным чартам и получать очки за свои ответы.

Для достижения цели были поставлены следующие задачи:
- реализовать веб-приложение на Flask;
- создать систему регистрации и авторизации;
- подключить базу данных для хранения пользователей и результатов;
- получить музыкальные данные из внешнего API;
- реализовать игровой процесс из нескольких раундов;
- добавить расчет очков;
- создать интерфейс выбора страны;
- реализовать таймер раунда;
- сохранить историю игр в профиле пользователя.

## 3. Описание функциональности

Пользователь может открыть главную страницу приложения и начать игру. Если пользователь зарегистрирован, его результаты сохраняются в профиле. Также приложение поддерживает игру без аккаунта, но в этом случае личная статистика не сохраняется.

Игра состоит из нескольких раундов. Количество раундов задается в конфигурации проекта. В текущей версии используется 3 раунда, в каждом раунде показывается 5 треков.

Во время раунда приложение загружает музыкальные треки для случайно выбранной страны. Пользователь слушает треки, затем выбирает предполагаемую страну из списка. Для удобства страны сгруппированы по регионам, а также доступен поиск по названию.

После отправки ответа система сравнивает выбранную страну с правильной и начисляет очки. По завершении всех раундов пользователь видит итоговый результат.

## 4. Используемые технологии

В проекте используются следующие технологии:

- Python - основной язык серверной части;
- Flask - веб-фреймворк для маршрутов, шаблонов и обработки запросов;
- Flask-Login - управление авторизацией пользователей;
- SQLAlchemy - ORM для работы с базой данных;
- SQLite - локальная база данных;
- Jinja2 - шаблонизация HTML-страниц;
- JavaScript - динамическая логика страницы игры;
- CSS - оформление интерфейса;
- requests - выполнение HTTP-запросов к внешним API;
- python-dotenv - загрузка переменных окружения;
- Apify Client - получение MP3-ссылок для треков;
- SoundCharts API - получение музыкальных чартов.

## 5. Структура проекта

Основная структура проекта:

- `app.py` - главный файл приложения. В нем создается Flask-приложение, описаны маршруты, логика игры, авторизация, работа с сессиями и обработка результатов.
- `db.py` - файл подключения к базе данных. Здесь создается SQLAlchemy engine и фабрика сессий.
- `db_models.py` - модели базы данных: пользователь, игра и раунд.
- `get_chart.py` - сервис для получения музыкальных данных из SoundCharts и MP3-ссылок через Apify.
- `country_neighbors.py` - логика определения соседних стран и начисления очков.
- `config.py` - настройки проекта: пути к файлам, количество раундов, время раунда, количество треков.
- `templates/` - HTML-шаблоны страниц.
- `static/` - статические файлы: CSS, JavaScript, аудиокэш.
- `data/` - JSON-файлы со списками стран, кодами стран, регионами и соседями.

## 6. Серверная часть

Серверная часть реализована с помощью Flask. Приложение обрабатывает маршруты главной страницы, регистрации, входа, выхода, профиля, старта игры, игрового раунда, ответа пользователя и итоговых результатов.

Для хранения состояния текущей игры используются Flask-сессии. В сессии сохраняются:
- активна ли игра;
- идентификатор игры;
- список стран для раундов;
- текущий номер раунда;
- текущий счет;
- результаты завершенных раундов.

Когда пользователь начинает новую игру, незавершенная предыдущая игра удаляется. Затем система выбирает случайные страны и создает запись игры в базе данных.

## 7. База данных и модели

В базе данных используется три основные таблицы.

### User

Модель пользователя хранит:
- логин;
- хэш пароля;
- никнейм;
- количество сыгранных игр;
- лучший результат;
- дату регистрации.

Также модель содержит методы регистрации, проверки пароля и получения истории игр.

### Game

Модель игры хранит:
- пользователя, если игра была сыграна авторизованным пользователем;
- итоговый счет;
- дату игры;
- статус завершения.

При завершении игры обновляется общий счет, количество игр пользователя и лучший результат.

### Round

Модель раунда хранит:
- номер раунда;
- правильную страну;
- выбранную страну;
- количество очков;
- тип совпадения.

Тип совпадения показывает, насколько ответ был близок: точное совпадение, соседняя страна, страна через одного соседа, неправильный ответ или отсутствие ответа.

## 8. Авторизация

Авторизация реализована через Flask-Login. Пользователь может зарегистрироваться, указав логин, никнейм и пароль. Пароль не хранится в открытом виде: перед сохранением используется хэширование.

При входе система ищет пользователя по логину и проверяет пароль. После успешного входа пользователь получает доступ к профилю, где отображается его статистика.

При выходе из аккаунта текущая незавершенная игра удаляется, а игровая сессия очищается.

## 9. Получение музыкальных данных

За получение музыкальных данных отвечает класс `SoundChartsService`.

Алгоритм работы:
1. По названию страны находится код музыкального чарта.
2. Выполняется запрос к SoundCharts API.
3. Из ответа извлекаются название трека, исполнитель, обложка и идентификатор песни.
4. Для песни ищется Spotify-ссылка.
5. Через Apify выполняется попытка получить MP3-ссылку.
6. Подготовленные данные передаются на страницу игры.

Дополнительно реализован локальный кэш аудио. Если аудиофайл уже был загружен раньше, приложение использует файл из кэша, а не скачивает его заново.

## 10. Игровая логика

В начале игры система случайно выбирает страны из доступного списка. Для каждой страны создается отдельный раунд.

Пользователь видит топ-5 треков, слушает их и выбирает страну. На ответ дается ограниченное время. Если время заканчивается, форма автоматически отправляется без выбранного ответа.

После ответа создается объект `Round`, который сохраняется в базе данных. Затем результат раунда добавляется в сессию, а счет игры увеличивается.

После последнего раунда игра завершается. Итоговый результат сохраняется в базе данных, а пользователь видит страницу результатов.

## 11. Система начисления очков

Очки начисляются не только за точный ответ, но и за географически близкий ответ.

Правила:
- 1000 очков - страна угадана точно;
- 700 очков - выбрана соседняя страна;
- 400 очков - выбрана страна через одного соседа;
- 0 очков - неправильный ответ или ответ не выбран.

Для этого используется файл со списком соседних стран. Функция определяет прямых соседей и соседей второго уровня. Такой подход делает игру интереснее, потому что близкие ответы тоже частично засчитываются.

## 12. Клиентская часть

Клиентская часть страницы игры реализована на JavaScript.

Основные функции:
- загрузка данных раунда через API-маршрут;
- отображение треков;
- предварительная загрузка аудио;
- запуск таймера;
- выбор страны;
- поиск по странам;
- автоматическая отправка ответа при окончании времени;
- просмотр обложки трека в модальном окне.

Важная особенность: таймер запускается только после того, как треки и аудио подготовлены. Это сделано для того, чтобы пользователь не терял время из-за загрузки данных.

## 13. Интерфейс

Интерфейс проекта состоит из нескольких страниц:
- главная страница;
- регистрация;
- вход;
- профиль;
- игровой раунд;
- результат раунда;
- итоговый результат;
- страницы ошибок.

В интерфейсе используются карточки, кнопки, иконки, темная и светлая тема. На странице игры страны сгруппированы по регионам, чтобы пользователю было удобнее искать ответ.

## 14. Основные трудности

Во время разработки возникли следующие трудности:

- Внешние API могут работать нестабильно или возвращать неполные данные.
- Не для каждого трека удается получить MP3-ссылку.
- Аудио нужно подготовить до старта таймера.
- Названия стран в разных источниках нужно сопоставлять между собой.
- Нужно было реализовать понятную и справедливую систему очков.
- Нужно было сохранять состояние игры между запросами.

Для решения этих проблем были использованы:
- обработка исключений;
- локальное кэширование аудио;
- отдельные JSON-файлы для стран, регионов и соседей;
- Flask-сессии;
- отдельные модели базы данных;
- отдельный класс для работы с музыкальным API.

Проект демонстрирует практическое применение Flask, SQLAlchemy, авторизации, внешних API, JavaScript и работы с данными.
