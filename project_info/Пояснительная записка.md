# Пояснительная записка к проекту Chart Detective

## 1. Введение

Chart Detective — веб-проект в формате музыкально-географической игры. Пользователь слушает популярные треки из музыкальных чартов разных стран и пытается определить, какой стране принадлежит текущий чарт.

Проект объединяет несколько направлений разработки: серверную часть на Python и Flask, работу с базой данных, авторизацию пользователей, интеграцию с внешними API, обработку JSON-данных и создание интерактивного интерфейса на JavaScript.

Главная идея проекта — использовать музыку как источник подсказок. Пользователь анализирует язык треков, имена исполнителей, стиль музыки и популярность песен, после чего выбирает страну. Правильный или географически близкий ответ приносит очки.

---

## 2. Цель и задачи проекта

Цель проекта — разработать веб-приложение, позволяющее пользователю угадывать страны по музыкальным чартам и получать очки за свои ответы.

Для достижения цели были поставлены следующие задачи:
- реализовать веб-приложение на Flask;
- создать систему регистрации и авторизации;
- подключить базу данных для хранения пользователей и результатов;
- получить музыкальные данные из внешнего API;
- реализовать игровой процесс из нескольких раундов;
- добавить расчёт очков с учётом географической близости ответа;
- создать интерфейс выбора страны с поиском и группировкой по регионам;
- реализовать таймер раунда;
- сохранять историю игр в профиле пользователя.

---

## 3. Описание функциональности

Пользователь может открыть главную страницу приложения и начать игру. Если пользователь зарегистрирован, его результаты сохраняются в профиле. Приложение поддерживает игру без аккаунта, но в этом случае личная статистика не сохраняется.

Игра состоит из нескольких раундов. Количество раундов задаётся в конфигурации проекта — в текущей версии 3 раунда, в каждом раунде показывается 5 треков.

Во время раунда приложение загружает музыкальные треки для случайно выбранной страны. Пользователь слушает треки, затем выбирает предполагаемую страну из списка. Для удобства страны сгруппированы по регионам и свёрнуты по умолчанию — можно раскрыть нужный регион или воспользоваться поиском по названию.

После отправки ответа система сравнивает выбранную страну с правильной и начисляет очки. По завершении всех раундов пользователь видит итоговый результат с разбивкой по каждому раунду.

---

## 4. Используемые технологии

В проекте используются следующие технологии:

- Python — основной язык серверной части;
- Flask — веб-фреймворк для маршрутов, шаблонов и обработки запросов;
- Flask-Login — управление авторизацией пользователей;
- SQLAlchemy — ORM для работы с базой данных;
- SQLite — локальная база данных;
- Jinja2 — шаблонизация HTML-страниц;
- JavaScript — динамическая логика страницы игры;
- CSS — оформление интерфейса, адаптация под мобильные устройства, тёмная и светлая темы;
- requests — выполнение HTTP-запросов к внешним API;
- python-dotenv — загрузка переменных окружения из `.env`-файла;
- Apify Client — получение MP3-ссылок для треков через Apify;
- SoundCharts API — получение музыкальных чартов.

---

## 5. Структура проекта

Основная структура проекта:

- `app.py` — главный файл приложения. В нём создаётся Flask-приложение, описаны маршруты, логика игры, авторизация, работа с сессиями и обработка результатов.
- `db.py` — файл подключения к базе данных. Здесь создаётся SQLAlchemy engine и фабрика сессий.
- `db_models.py` — модели базы данных: пользователь, игра и раунд.
- `get_chart.py` — сервис для получения музыкальных данных из SoundCharts и MP3-ссылок через Apify.
- `country_neighbors.py` — логика определения соседних стран и начисления очков.
- `config.py` — настройки проекта: пути к файлам, количество раундов, время раунда, количество треков.
- `templates/` — HTML-шаблоны страниц.
- `static/` — статические файлы: CSS, JavaScript, кэш изображений и аудио.
- `data/` — JSON-файлы со списками стран, кодами чартов, регионами и соседями.

---

## 6. Серверная часть

Серверная часть реализована с помощью Flask. Приложение обрабатывает маршруты главной страницы, регистрации, входа, выхода, профиля, старта игры, игрового раунда, ответа пользователя и итоговых результатов.

Для хранения состояния текущей игры используются Flask-сессии. В сессии сохраняются:
- активна ли игра;
- идентификатор игры;
- список стран для раундов;
- текущий номер раунда;
- текущий счёт;
- результаты завершённых раундов.

Когда пользователь начинает новую игру, незавершённая предыдущая игра удаляется из базы данных и сессии. Затем система выбирает случайные страны и создаёт запись игры в базе данных.

Данные о треках передаются клиенту через отдельный API-маршрут `/api/game/data` — браузер запрашивает их асинхронно сразу после загрузки страницы игры. Это позволяет показать экран загрузки с прогрессом, не задерживая открытие страницы.

Также реализован маршрут `/api/game/abort`: если пользователь закрывает вкладку или уходит в середине игры, браузер автоматически отправляет на этот адрес сигнал через `navigator.sendBeacon`, и незавершённая игра удаляется.

## 7. База данных и модели

В базе данных используется три основные таблицы.

### User

Модель пользователя хранит:
- логин;
- хэш пароля;
- никнейм;
- количество сыгранных игр;
- лучший результат;
- дату регистрации.

Также модель содержит методы регистрации, проверки пароля и получения истории игр.

### Game

Модель игры хранит:
- пользователя, если игра была сыграна авторизованным пользователем;
- итоговый счёт;
- дату игры;
- статус завершения.

При завершении игры обновляется количество игр пользователя и его лучший результат.

### Round

Модель раунда хранит:
- номер раунда;
- правильную страну;
- выбранную страну;
- количество очков;
- тип совпадения.

Тип совпадения показывает, насколько ответ был близок: точное совпадение, соседняя страна, страна через одного соседа, неправильный ответ или отсутствие ответа.

---

## 8. Авторизация

Авторизация реализована через Flask-Login. Пользователь может зарегистрироваться, указав логин, никнейм и пароль. Пароль не хранится в открытом виде: перед сохранением используется хэширование.

При входе система ищет пользователя по логину и проверяет пароль. После успешного входа пользователь получает доступ к профилю, где отображается его статистика и история игр.

При выходе из аккаунта текущая незавершённая игра удаляется, а игровая сессия очищается.

---

## 9. Получение музыкальных данных

За получение музыкальных данных отвечает класс `SoundChartsService` в файле `get_chart.py`.

Алгоритм работы метода `get_tracks()`:

1. По названию страны из JSON-файла определяется код музыкального чарта.
2. Выполняется запрос к SoundCharts API — возвращается список топ-5 треков (название, исполнитель, идентификатор).
3. Параллельно (через `ThreadPoolExecutor`) для каждого трека скачивается обложка альбома и запрашивается Spotify-ссылка.
4. Через Apify по Spotify-ссылке получается прямая MP3-ссылка.
5. Треки, для которых не удалось получить MP3, исключаются из результата. Их обложки при этом сразу удаляются из кэша, чтобы не занимать место.
6. Подготовленные данные возвращаются в приложение и отдаются клиенту.

Аудиофайлы кэшируются локально: если аудио для трека уже было скачано ранее, приложение использует файл из кэша. После завершения раунда все файлы раунда — аудио и обложки — удаляются.

---

## 10. Игровая логика

В начале игры система случайно выбирает страны из доступного списка. Для каждой страны создаётся отдельный раунд.

Пользователь видит топ-5 треков, слушает их и выбирает страну. На ответ отведено ограниченное время — 30 секунд. Если время заканчивается, форма автоматически отправляется без выбранного ответа.

После ответа создаётся объект `Round`, который сохраняется в базе данных. Результат раунда добавляется в сессию, счёт игры увеличивается, и пользователь видит страницу с результатом раунда.

После последнего раунда игра завершается. Итоговый результат сохраняется в базе данных, а пользователь переходит на страницу итогов.

---

## 11. Система начисления очков

Очки начисляются не только за точный ответ, но и за географически близкий ответ.

Правила:
- 1000 очков — страна угадана точно;
- 700 очков — выбрана соседняя страна;
- 400 очков — выбрана страна через одного соседа;
- 0 очков — неправильный ответ или ответ не выбран.

Для этого используется JSON-файл со списком соседних стран. Функция в `country_neighbors.py` проверяет прямых соседей, а при необходимости — соседей второго уровня. Такой подход делает игру интереснее, потому что близкие географически ответы тоже частично засчитываются.

## 12. Клиентская часть

Клиентская часть страницы игры реализована на JavaScript в файле `game.js`.

При загрузке страницы сразу показывается экран загрузки с тремя шагами: «Запрос чарта», «Треки и обложки», «Аудио». Браузер отправляет запрос к `/api/game/data`, и по мере получения ответа шаги последовательно загораются зелёным. Если загрузка занимает слишком долго (более 2 минут), появляется сообщение с обратным отсчётом до автоматической перезагрузки страницы.

После получения данных экран загрузки скрывается, и пользователь видит игровое поле.

Основные функции JavaScript:
- отрисовка карточек треков с аудиоплеером, обложкой, названием и исполнителем;
- запуск обратного отсчёта таймера;
- выбор страны из списка (клик по кнопке страны);
- поиск стран по названию с автоматическим раскрытием групп;
- переключение групп стран по регионам (клик по заголовку группы);
- автоматическая отправка ответа при истечении времени;
- просмотр обложки трека в модальном окне;
- очистка игры при уходе со страницы через `sendBeacon`.

Важная особенность: таймер запускается только после того, как все данные получены и отрисованы. Это гарантирует, что пользователь не теряет время из-за загрузки.

---

## 13. Интерфейс

Интерфейс проекта состоит из нескольких страниц:
- главная страница с описанием игры и кнопкой старта;
- регистрация и вход;
- профиль с общей статистикой и историей игр;
- игровой раунд;
- результат раунда;
- итоговый результат;
- страницы ошибок (404, 500).

Интерфейс поддерживает тёмную и светлую темы. Переключатель находится в шапке, выбранная тема сохраняется в `localStorage`. Дизайн адаптирован для разных устройств: на мобильных треки и выбор страны располагаются вертикально, аудиоплеер занимает всю ширину, а список стран прокручивается вместе со страницей.

---

## 14. Основные трудности

Во время разработки возникли следующие трудности:

- **Нестабильность внешних API.** SoundCharts и Apify могут возвращать неполные данные, неожиданный формат ответа или ошибку. Для каждого из этих случаев добавлена обработка исключений и запасное поведение.
- **Отсутствие MP3 для части треков.** Не для каждого трека удаётся получить аудио. Такие треки фильтруются, а их обложки сразу удаляются из кэша.
- **Очистка кэша.** Скачанные аудиофайлы и обложки нужно удалять своевременно. Реализована очистка после каждого раунда, при старте новой игры и при аварийном выходе.
- **Сопоставление названий стран.** В разных источниках одна и та же страна может называться по-разному. Для согласования используются отдельные JSON-файлы соответствий.
- **Состояние игры между запросами.** Нужно было сохранять текущий раунд, счёт и список стран между несколькими HTTP-запросами — реализовано через Flask-сессии.
- **Синхронизация загрузки и таймера.** Таймер не должен запускаться до того, как треки готовы. Решено через асинхронную загрузку данных и запуск таймера только после получения ответа.

## 15. Результат

В результате разработано рабочее веб-приложение Chart Detective. Пользователь может зарегистрироваться, начать игру, слушать треки, выбирать страну, получать очки и просматривать свои результаты.

Проект демонстрирует практическое применение Flask, SQLAlchemy, авторизации, внешних API, кэширования файлов, JavaScript и адаптивного интерфейса в рамках одного связного приложения.

## 16. Возможные направления развития

В будущем проект можно существенно расширить:

- **Интерактивная карта.** Вместо списка стран пользователь кликает прямо по карте мира — это сделает игровой процесс значительно нагляднее и интереснее.
- **Таблица лидеров.** Глобальный рейтинг игроков по набранным очкам, чтобы добавить соревновательный элемент.
- **Соревновательный режим.** Возможность сыграть против другого пользователя в реальном времени или в формате синхронных раундов.
- **Уровни сложности.** Например, лёгкий уровень — крупные страны со стабильными чартами; сложный — маленькие или малоизвестные страны.
- **Выбор количества раундов и времени.** Позволить пользователю самому настраивать длину и темп игры.
- **Режимы по континентам.** Игра только по Европе, Азии, Латинской Америке и т.д.
- **Подсказки.** За штрафные очки пользователь может открыть подсказку — например, первую букву названия страны или регион.
- **Расширенная статистика.** Показывать, какие страны угадываются чаще всего, процент правильных ответов, динамику результатов.
- **Дополнительные источники данных.** Подключить другие музыкальные сервисы, чтобы снизить зависимость от одного API.
- **Улучшенная обработка недоступности API.** Добавить резервные источники или заранее подготовленные наборы треков для бесперебойной работы.

## 17. Заключение

Chart Detective — это проект, который показывает, как можно объединить музыку, географию и веб-разработку в одном приложении. Реализованы игровая механика, работа с пользователями, база данных и интеграция с внешними сервисами.

В ходе работы над проектом был получен практический опыт построения полноценного веб-приложения: от серверной логики и авторизации до работы с внешними API, кэшированием файлов и интерактивным интерфейсом.
